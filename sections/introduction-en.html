Introduction back to ToC

The AffectedBy ODP consists of two classes aff:FeatureOfInterest and aff:Quality; and three properties aff:influencedBy, aff:belongsTo, and aff:affectedBy.

The AffectedBy ODP defines the aff:belongsTo object property to define the relationship between qualities and features of interest. It is defined as functional to support the notion that a quality is intrinsic to the feature of interest to which it belongs, thus following the conceptualization in DUL. Additionally, the following axiom formalizes that every quality belongs to a feature of interest:
aff:Quality ⊆ ∃aff:belongsTo.aff:FeatureOfInterest

Besides, the AffectedBy ODP introduces the aff:affectedBy object property to relate a quality that is affected by another quality and it is declared to be transitive.

The aff:influencedBy object property is also introduced to relate features of interest and qualities. The next property chain axiom is included in its definition:
aff:influencedBy ⚬ aff:affectedBy ⊆ aff:influencedBy

Finally, the property axiom representing that the aff:belongsTo object property is a subproperty of the inverse of the aff:influencedBy object property is introduced:
aff:belongsTo ⊆ inverse(aff:influencedBy)

Axioms included in the AffectedBy ODP provide inferences that allow answering the formulated CQs properly. Note that only triples with predicates aff:belongsTo and aff:affectedBy need to be asserted, the remaining triples are inferred with the provided property axioms.

Finally, the object chain axiom aff:influencedBy o aff:affectedBy rdfs:subPropertyOf aff:influencedBy is asserted.
